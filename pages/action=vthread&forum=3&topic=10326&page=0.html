Process Information & Killer
После запуска ехе нужно нажать Ctrl+Shift+~ , чтобы окошко программы появилось.

История версий:

Версия 1.0 alpha
Возможности:
-просматривать и убивать любые процессы (в т.ч. и системные)
-изменять приоритет процесса
-просматривать свойства процессов и дополнительно ехе файла программы (версия, описание)
-просматривать загруженные библиотеки для каждого процесса
-настройки приоритета, скорости обновления и автозагрузка программы
-программа при нажатии кнопки закрыть просто пропадает из виду, чтобы вызвать её нажмите Ctrl+Shift+~
-использована низкоуровневая обработка нажатий на клавиатуру для мгновенного вызова, если какая-то игрушка или прога повесила комп.
-процессы сортируются в обратном порядке, поэтому последний запущенный процесс будет сверху, и при вызове курсор попадёт именно туда, для того чтобы убить процесс остаётся нажать del
-возможен запуск только одной копии программы
Все вопросы и предложения направлять на e-mail:multiarc@tut.by

Версия 1.0 beta
[+] Список процессов обновляется мгновенно, после убивания процесса.

Версия 1.1
[-] Сообщения об ошиках в API (OpenProccess, OpenProccessToken, etc.)
[*] Исправлены некоторые ошибки
[+] Возможность внедрять любые dll в процессы
[+] Возможность выгружать dll из процесса (будьте осторожны с этой возможностью, можно потерять процесс...)
[+] Информация о коммандной строке процесса
[+] Возможность перезапуска процесса (с теми же парамтрами) (себя перезапускать нельзя)

Версия 1.2 (18.08.2007)
[*] Исправлены некоторые ошибки
[+] Индикатор загрузки процессора
[+] Использование памяти, файла подкачки, свободная память, свободная память файла подкачки
[+] Приоритеты процессов "Выше нормального" и "Ниже нормального"

Версия 1.2.1 (19.08.2007)
[*] Изменён индикатор загрузки
[+] Опция цвета индикатора загрузки

Версия 1.2.2 (20.08.2007)
[*] Изменён индикатор загрузки

Версия 1.3 (21.08.2007)
[+] Добавлена возможность приостановки/возобновления процесса

Версия 1.3.1 (21.08.2007)
[*] Исправлен визуальный баг при приостановке любого процесса

Версия 1.4 (24.08.2007)
[+] Если в убиваемый процесс перехватывает TerminateProcess, реализован альтернативный метод убивания через ExitProcess

Версия 1.5 (05.10.2007)
[*] Исправлено неправлиное определение размера памяти, если её много
