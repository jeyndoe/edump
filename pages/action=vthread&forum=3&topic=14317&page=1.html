Code:
  1. Version 3.90 beta 1
  3.    1. WinRAR version for Windows x64 is available. If you use
  4.       Windows x64, it is strongly recommended to install 64 bit
  5.       WinRAR version. It provides a higher performance and better 
  6.       shell integration than 32 bit version.
  8.    2. RAR compression speed is improved for multi-core
  9.       and multi-CPU systems. This improvement is most noticeable
  10.       in Windows Vista and Windows 7 operating systems.
  12.    3. "Remove duplicate folders from extraction path" option
  13.       in "Settings/Compression" dialog is replaced by more universal
  14.       "Remove redundant folders from extraction path" option.
  15.       This option will eliminate redundant archive name based folders
  16.       from extraction path if you unpack an archive with
  17.       "Extract to DestName" context menu command and if archive
  18.       root folder contains only one folder and no files.
  20.    4. Changes in "Extraction path and options" dialog:
  22.       a) "New folder" button creates a new subfolder in currently
  23.           selected folder;
  25.       b) F2 key renames a selected folder in the folders tree;
  27.       c) F5 key updated the tree pane contents;
  29.       d) Del key removes a selected folder in the folders tree.
  31.    5. You can enable "Show seconds" option in "Settings/File list"
  32.       dialog if you wish to see seconds in file dates in file list
  33.       in WinRAR shell.
  35.    6. "Where to check for SFX archives" options group
  36.       in "Settings/Integration/Context menus items" dialog lets
  37.       you to control processing of SFX archives in context menus.
  38.       For example, if you frequently right click ".exe" files
  39.       on slow network disks, you can turn off "Network disks" options
  40.       to minimize the delay before displaying the context menu.
  42.    7. If you sort files by name in the file list in WinRAR shell,
  43.       WinRAR will use the new logical file name sorting,
  44.       same as in Windows Explorer, considering digits in file names
  45.       by their numerical value. So files will be sorted as
  46.       1.txt, 2.txt, 10.txt instead of previous 1.txt, 10.txt, 2.txt.
  47.       This new sort behavior is available in Windows XP Service Pack 2
  48.       and newer.
  49.    
  50.    8. Ctrl+W key combination can be used to close the main WinRAR window
  51.       also as WinRAR viewer windows. "View as Windows text" shortcut
  52.       in WinRAR viewer changed from Ctrl+W to Ctrl+I.
  53.    
  54.    9. New command line switch -r- disables recursion completely.
  55.       So 'rar a -r- arc dirname' command will add only the empty
  56.       dirname folder and ignore its contents. By default, if dirname
  57.       does not include wildcards, RAR adds its contents even if
  58.       -r switch is not specified.
  60.   10. If used when extracting, the new command line switch -ai forces
  61.       RAR to ignore file attributes. When using this switch,
  62.       extracted files will always have attributes assigned by
  63.       operating system to a newly created file by default.
  65.   11. If output file name is not specified in "cw" command, console RAR
  66.       will send comment data to stdout.
  68.   12. When compressing stdin data with -si[name] switch, RAR sets 
  69.       modification time of archived entry to current system time.
  70.       Previous RAR versions did not fill this field, resulting in
  71.       meaningless modification time for stdin.
  73.   13. Message displayed when you place the mouse cursor on WinRAR
  74.       tray icon includes the archive name now. Previously only
  75.       time left and total percent were displayed for archiving operations.
  76.   
  77.   14. Bugs fixed:
  79.       a) WinRAR could fail to open tar or tar.gz archive if such archive
  80.          contained a file larger than 8 GB;
  81.       
  82.       b) WinRAR context menu did not work properly in Windows 7 beta
  83.          if icons in context menus were enabled and user clicked
  84.          a file inside of Windows 7 Library folder;
  85.   
  86.       c) previous WinRAR versions failed to rename files having 5 or more
  87.          continuous spaces in the name. WinRAR shell does not display
  88.          such spaces for safety reasons, because they can hide an actual
  89.          file extension. But this security measure prevented the rename
  90.          command to work properly.
