StarForce Technologies - developer of copy protection software announces release of StarForce FrontLine 4.0 – a new version of copy protection system.

Protection possibilities of FrontLine have been significantly broadened. FrontLine 4.0 has basic protection for 64 bit applications. This is the first step to fully functional 64bit application protection in future versions.

Another major change is the protection of services. This implies protection of applications like anti-viruses and etc.

FrontLine 4.0 is much more effective in controlling licensed applications and distribution chain. New DRM technologies provide a new spectrum of opportunities to control the usage of protected application.

The concept of “license” is different in FrontLine 4.0. A number of licenses may be saved for each protected product there on one computer. The client can choose which license to use for starting the protected application. The choice of license is defined by the serial number.

The activation protocol via proxy and firewalls has been improved. The activation mechanism changed significantly. The time on the user’s PC is no longer used during activation. This allows to avoid many mistakes. Protection against clock backward setting is implemented in the new version. That prevents usage of products with expired licenses. FrontLine 4.0 also has new opportunities in adding sessions of serial number generations to the black list. Generating unlimited number of serial numbers within one session is now possible as well.

FrontLine 4.0 also has the following important changes: the usage of protected application can be limited by number of star-ups and/or by run time down to a minute’s precision, the keys’ validity can be limited down to a minute’s precision as well.

Support of multi-user licenses has been upgraded too. One serial number can now be used by several computers while the number of re-activations is counted separately. FrontLine 4.0 has the option of external initialization control. The new version allows clients to create applications that will control the processes of protection initialization and activation.

SOAP-interface support on activation server has been added. This option allows clients to integrate with the DRM system. Clients can now set the date of product’s activation, change the parameters of separate serial numbers and generations and generate rescue keys on the activation site.

FrontLine 4.0 has high resistance to cracking and emulation. Protection against emulators and filters has been improved, protection against Daemon Tools 4.0 added as well.

В FrontLine 4.0 can now protect DVDs based on Read DVD Information. Any writable discs have a special ATIP area. This area can be read by any CD-RW and DVD drives, including non-writable. Disc binding based only on Read DVD Information checks only the type of disc by reading the ATIP. The type of disc can be defined by both reader and writer type of drives.

Undoubtedly the major improvement is the service of automated StarForce FrontLine drivers’ uninstall. This change was made because of multiple end-user requests. From now on the drivers will be removed automatically after the uninstall of all StarForce protected applications.

We confidently state that StarForce FrontLine 4.0 is one of the most reliable and user-friendly solutions in the copy-protection market.

StarForce experts will present the latest version of copy protection at CeBIT 2006.
