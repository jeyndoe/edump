Chrome сделал первый шаг к FireFox - стал поддерживать плагины

Компания Google исправила один из самых серьезных недостатков своего нового браузера Chrome - отсутствие поддержки плагинов-расширений. Именно поддержка плагинов выдвинула Firefox на лидирующие позиции среди программ для веб-серфинга. Недавно на блоге программиста Google Аарона Будмана (Aaron Boodman) появилась информация о том, что Google скоро исправит этот недочет и в Chrome появится поддержка плагинов-расширений. Также на блоге Будмана можно ознакомиться с проектной документацией под названием Extensions (расширения).

В первую очередь Google сделает свободный доступ к инструментам разработки плагинов, чтобы расширения стали доступны для внесения корректив другими разработчиками. Затем Google просто установит плагины. Новые расширения для Chrome будут удобны в обращении благодаря настройкам в виде панелей и возможностью изменения прав доступа.

Обновление плагинов для Chrome будет происходить фоновом режиме. Для повышения безопасности браузера их изолируют. В первую очередь планируется создать расширение для блокировки рекламы. Несомненно, в спектр функций плагинов будут включены фильтрами веб-контента и родительский веб-фильтр.

Созданием плагинов разработчики пытаются приблизить Chrome к своему основному конкуренту Firefox. В дальнейшем планируется оснащение Chrome другими функциями, которые уже имеются у популярных браузеров.

Google Chrome: безопасность браузера под вопросом

Ричард Чапин (Richard Chpin) опубликовал на сайте компании Chapin Information Services результаты проверки функционала менеджера паролей в популярных браузерах. Google Chrome разделил последнее место с safari.

Компания Chapin Information Services (CIS) сообщила Google о критических уязвимостях в браузере Chrome во время бета тестирования приложения. Ни одна их существующих уязвимостей не была устранена в финальной версии браузера.

Среди других проблем, три уязвимости, которые позволяют злоумышленнику получить доступ к сохраненным паролям в браузере без взаимодействия с пользователем:


1. Отсутствие проверки назначения отправляемых паролей
2. Отсутствие проверки страниц, запрашиваемых пароли
3. Невидимые элементы форм могут взаимодействовать с менеджером паролей

Метод, описанный компанией 2 года тому назад, позволяет эксплуатировать эти уязвимости без выполнения клиентских сценариев. Эти три уязвимости совместно с остальными 17, которые были также обнаружены в механизме управления паролями в Google Chrome, представляют серьезную угрозу личным данным пользователей.

В тестировании принимали участие Opera 9.62, Firefox 3.0.4, Internet explorer 7, Safari 3.2 и Google Chrome 1.0. Самые лучшие результаты показали Opera и Firefox, которые успешно прошли 7 из 21 тестов.

Результаты тестирования доступны по адресу: www.info-svc.com/news/2008/12-12/

Источник: securitylab.ru
