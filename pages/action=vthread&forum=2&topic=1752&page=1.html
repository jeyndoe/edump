Myp1blka пишет:
Но все же в чем принцип метода выдирания ..?

Та какой там прицнип, всё просто. Находишь оверлей, сама книга в прицнипе начинается с такой сигнатуры *M2E*, далее ищешь сигнатуру начала zip файла: 504B0304 (PK), записываешь на винт всё до следующей такой сигнатуры с расширением zip.
Я искал вот так:

function FindSign(Beg: pointer): pointer;
begin
Result := 0;
while dword(Beg) < EOF do begin
if dword(Beg^) = $04034B50 then begin
Result := Beg;
break;
end;
inc(dword(Beg));
end;
end;


Распаковываешь любым путём, в архиве будет файл с каким-то левым названием, ПЕРЕИМЕНОВЫВАЕШь его в соответствии с порядковым номером начиная с 1. Т.е. если ты нашла первую сигнатуру, то это будет файл №1, вторую сигнатуру, то это файл №2 и etc.
После подставляешь расширение в соответствии с сигнатурой распакованного файла.
Далее, все картинки и звуки на страницах при этом пропадут, потому что в html файлах ссылки на них выглядят таким образом: ?[порядковый номер файла без расширения]. Остаётся заменить все такие конструкции на пути к соответствуюзим файлам и всё.

Если это exebook, то ломать там нечего - ничего кроме картинок в неважном качестве внутри не содержится.

Ключ -i нужен только чтобы предотвратить распаковку. В вашем случае он безразличен.

Если запускаете как

ebu ebook.exe, то рядом должен появиться файлик с именем ebook.exe.info с информацией о файле.

В вашем случае, там должно быть примерно так:
Code:
  1. File >>    10382848  2013-02-08  14:45:32  $B502C3B0  |  .\ebook.exe
  2. $001 >> 23D7B5B67D307593460FBC3333433C48  |  Stub: $009E6E00  Over: $00000000
  3. Stub >> Isn't an e-book
  4. Envl >> Publicant (ExeBook)  [v. <$3A>]
