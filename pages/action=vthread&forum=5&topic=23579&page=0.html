Здравствуйте уважаемые форумчане. Есть одна строительная программа, она работает с ключом Hasp. На ключе написано Hasp HL, ключик зелёненький такой USB.
Название AВC версия 5.3. Ключик с дампил с помощью программы Toro Aladdin Dongles Monitor. Другие дамперы не создавали нормальный дамп. h5dmp сказал, что ключик не из его группы и не делала дамп.
h4dmp вроде бы и создал дамп, но был только один файл дампа hasp.dmp. конвертнул полученное через UniDumpToReg для эмулятора Chingachguk based Hasp HL, запустил с помощью vusbbus и тут же при запуске отловил ошибку. Чем только не дампил, даже HASP Hardlock dongle 2007-2010 и везде ловил ошибку при запуске программы. Потом с дампил с помощью Toro Aladdin Dongles Monitor получил 2 файла hasp.dmp и hhl_mem.dmp конвертнул полученное, запустил эмулятор и программа запустилась без каких либо ошибок, работала, но выползли подводные камни о которых я не знал. Одна из подпрограмм AВCInfo после открытия выдаёт ошибку "Отсутствие лицензионного файла или файл базы поврежден". Вырубаю эмулятор программа не запускается вообще. По идее эмулятор работает, но почему то не до конца. Программа написана на Delphi.
Скриншот Die Прилагается.
Теперь вопрос:
1. В чём может быть проблем и как её решить?
2. Можно ли Распаковать этот файл AВCInfo с помощью инвалидного эмулятора?
