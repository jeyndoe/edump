[img]There are two types of algo in SSPro keys - standard and encanced.
Standard algorithm is related with Write password. So, if Write password is known, only 2^28 bruteforce over single 8-bytes query is required.

Enchenced algorithm has design flaw that allows to bruteforce all 45 bits of secrets in couple of minutes if results of querying of about 4KBytes (over single algo cell) is available.
There are some chances that searching for algo descriptor values could be performed more efficient - at least backup utility from safe-key.com does not collects too much query data...[/img]

can some one talk more about the topic please.
