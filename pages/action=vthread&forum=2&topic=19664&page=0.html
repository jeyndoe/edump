В общем сам файл лицензии представляет из себя zip архив, в котором лежат файлы: HardID, Author, Company, EMail.
Первые 6 байт файла заменены на licv01, после этого зашифровано ГОСТом..

При валидации лицензии файл расшифровывается, помещается в массив байт, проверяются первые 6 байт, если они равны licv01, то массив копируется в промежуточный,
который меньше на 4 байта, т.е. представляет из себя массив байт, где вместо licv01 в начале просто 01. Дальше первые байты (эти самые 01) меняются на PK,
т.е. восстанавливается нормальный zip из которого после читаются HardID, Author, Company, EMail.

HardID - генерируется с помощью программы (hid.exe) из ресурсов FCETools.dll, после обрабатывается функцией ConvertToDigit в ней ConvertToDigit_Inner. HardID после
этих функций - это то, что мы видим в окне Registration: Reg Code вида xxxx-xxxx-xxx-xxxx-xxxx

Author всегда должен быть Sharman Staples.

В общем лениво было переписывать самому все это дело, поэтому генератор будет работать, если его поместить в папку с программой, т.к. использует FCE.dll и FCETools.dll.

Работает на .NET Framework 4, ругнется, если такового нет )

Если что-то не понятно, спрашивайте, помогу, чем смогу
