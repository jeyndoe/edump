Есть программка Database Comparer for SQL Server (V3.2.0.5).

Запакована: ASProtect 1.2x - 1.3x [Registered] -> Alexey Solodovnikov
Написана на Delphi6-7.

Интересует именно проверка ключа.

Данные по исследованию:

Процедура проверки ключа имеет следующий вид:

Начинается по адресу 0083F10C

Code:
  2. ...
  3. 0083F165  CALL MsCompar.004051FC Берем длинну ключа
  4. 0083F16A  CMP EAX,5 Должен быть больше 5 символов
  5. 0083F16D  JLE MsCompar.0083F220
  6. 0083F173  MOV EAX,DWORD PTR SS:[EBP-4]
  7. 0083F176  CMP BYTE PTR DS:[EAX+4],2D  Сравниваем 5ый символ - должен быть "-"
  8. 0083F17A  JNZ MsCompar.0083F220
  9. ...
  10. 0083F1F8  CALL MsCompar.0040D228 Приводит первые 4 символа ключа к инту (StrToInt) Не понял зачем
  11. ...
  12. 0083F219  CALL MsCompar.00404944 Вываливаемся сюда по ексепшену
  13. 0083F21E  JMP SHORT MsCompar.0083F26F и перепрыгиваем
  14. ...
  15. 0083F26F  PUSH 0 Далее идет (насколько я понимаю) несколько проверок ключа.
  16. ...
  17. 0083F27B  CALL MsCompar.0083F0E4 тут
  18. ...
  19. 0083F298  CALL MsCompar.0083F0F4 тут
  20. ...
  21. 0083F2C2  CALL MsCompar.0083F0FC и вот тут
  22. 0083F2C7  TEST AL,AL 
  23. 0083F2C9  JE SHORT MsCompar.0083F2DB отсюда выпрыгиваем в инвалид кей


При генерации ключа (как мне кажется) используется 3 параметра:
1) имя регистрации
2) дата (так как ключ имеет срок действия)
3) тип использования:
Non-commercial license
Business license
Enterprise license
Site license
Not For Resale license

Самому разобраться не удается т.к. недостаточно опыта в асме.
