Последнее время как грибы после дождя стали появляться новые технологии
виртуализации программ. До недавнего времени мог назвать только одно
решение такого рода Softricity Softgrid (которому уже около 4-х лет)
эту компанию и технологию, почуяв лакомый кусок, как всегда купила Microsoft.
Некоторые проеты еще проходят бета-тестирование (как Thinstall Virtualization и
Citrix Tarpon), некоторые уже используются (Altiris SVS).

Вобщем в этой теме хотелось бы осуждать все что связано с существующими, и
особенно новыми (уверен что появятся еще) технологиями виртуализации.

Под виртуализацией здесь понимается не виртуальные машины вроде VMware и VirtualPC,
которые эмулируют целые операционные системы, а виртуализация отдельных элементов
нужных программам для работы (ключей реестра, динамически подключаемых библиотек,
файлов, и т.п.)


Пока эти решения делятся на 2 категории: для IT-специалистов и обычных пользователей.

Для первых обещают легкость развертывания нужного софта на компьютеры пользователей,
ведь не требуется установки на десятки или сотни компов, установка и настройка производится только один раз.

Для вторых - пока только переносимость установленных программ, данных и настроек на
съемных носителях. (к этой категории отноятся только MojoPac, Ceedo, U3 )

Далее привожу все что удалось накопать мне.

ЗЫ Жалко что посты редктировать можно только 3 часа, просто хотелось бы вносить
изменения и особенно русские описания. Но думаю модераторы помогут.

2. Altiris Software Virtualization Solution

--------------------------------------------------
офсайт www.altiris.com/Products/SoftwareVirtualizationSolution.aspx

страница с готовыми вртиуализированными прогами, их обсуждение и т.п. juice.altiris.com/svs

страница загрузки www.altiris.com/Download/download.aspx



Основу функционирования SVS составляют так называемые виртуальные пакеты (VSP, Virtual Software Package), представляющие собой наборы файлов и ключей реестра. Один из самых простых и очевидных способов их получения – мониторинг стандартной процедуры установки ПО, который осуществляется из административной консоли SVS. Пакеты можно создавать и вручную, причем в них допускается размещать и данные (в том числе и исключительно их). При этом VSP хранятся в специальной папке, минимально защищенной от посторонних глаз. Безусловно, разработчики могли бы воспользоваться и какими-то более серьезными механизмами в духе rootkit, однако до сих пор из подобных экспериментов ничего хорошего не выходило. Кроме того, пакеты можно превращать в архивы (VSA, Virtual Software Archive) для транспортировки на другие машины. Собственно, подобный метод распространения ПО является одной из основных задач SVS, а дополнительная переносимость обеспечивается благодаря всемерному использованию системных переменных (вроде %windir%).

Независимые слои позволяют мирно сосуществовать нескольким версиям одного и того же файла (причем так, будто бы они находятся в одном и том же месте), т. е. им, в принципе, не страшны явления вроде DLL Hell. Благодаря этому можно:

* спокойно экспериментировать с любыми обновлениями или системными заплатками. Деактивация соответствующего слоя гораздо проще и эффективнее стандартных механизмов System Restore;
* параллельно использовать разные версии одного и того же ПО, не боясь нарушить их работоспособность (для целей тестирования, обучения пользователей и т. д.);
* работать с заведомо несовместимыми между собой (практически по любым причинам) прикладными программами, в том числе одновременно;
* обеспечить относительно защищенную среду для выполнения неизвестных приложений и интернет-программ, в первую очередь браузеров (на уже упоминавшемся ресурсе Altiris Juice придуман даже способ виртуализации Internet Explorer). В этом случае весь загруженный контент, в частности ActiveX-элементы и spyware, остается внутри слоя и может быть легко уничтожен.

6. Ceedo ( PowerToGo )

офсайт ceedo.com/



Ceedo - a de-facto standard for portable computing: Ceedo introduces a straightforward, proven and high-end platform for carrying and operating applications on a portable storage device, and launching them on any host computer. The platform manages the applications’ launching and operation in a completely virtual and independent environment, allowing them to run regardless of the specific settings of the host computer.

True application independence: A program could be launched from an entirely alien computer, one which has never before been introduced to that application. The launch is performed immediately, without delay required to copy files or register components, and while running acts as if the working environment is that of the host computer itself.

Leaving no trace: The Ceedo platform, combined with any type of removable media, creates the ultimate application portability mechanism; applications are now cut loose from the bonds that tie them to a singular operating system and working environment, and are now free to be carried and operated anywhere, anytime.

7. U3

офсайт www.u3.com/




Технология U3 позволяет производителям создавать портативные приложения, которые могли бы легко выполняться на сменных носителях, не оставляя информации на компьютере. Использования программ виртуализации для запуска разного софта без конфликтов и необходимости установки конечно хорошо. Но хотелось бы большего - на основной ОС стоит софт которому необходима прямая работа с оборудованием или R0, и устанавливается несколько полноценных виртуализированых сред (с легкой виртуализацией в R3) заточенных под определенную задачу: офис, вардрайвинг, реверсинг, инет (браузеры, IM, P2P), работа с графикой, обработка звука, сервера (FTP, HTTP), и т.д. Переключение между VM организованное как Multi Desktop. Переносить такие VM как набор файлов без необходимости предварительной настройки.

Систему такого типа теоретически можно организовать с помощью Virtuozzo. Но Virtuozzo заточен под VPS, и есть слишком много недостатков: слишком тяжелая и неповоротливая программа (несколько ГБ после установки), неудобное управление (для домашнего использования), доступ к виртуальным машинам только через сеть (Remote Desktop, Radmin), только Windows 2003 Server. В тоже время создать систему на базе Thinstall/Altiris/SandboxIE весьма проблематично из-за того, что эти программы сделаны для запуска нескольких софтин в VM, а не предназначены для виртуализации полного набора ОС в R3.

Может кто знает что-нибудь среднее между Virtuozzo и Thinstall, на базе чего можно сделать вышеописанную систему?
