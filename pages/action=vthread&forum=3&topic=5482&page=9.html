mihalas, все несколько проще. Для распаковки любой программы, которая не хочет извлекаться в реальную файловую систему, нужно в папке с исполняемым файлом программы, в данном случае Wings 3D.exe, создать папку Thinstall, все изменения, производимые программой, будут копироваться в нее. Если эту папочку не создавать, то все хранится в C:\Documents and Settings\Название профиля\Application Data\Thinstall. Можно заранее обеспечить работу в реальной файловой системе двумя способами:
1. Создать файл ##Attributes.ini с содержанием
[Isolation]
DirectoryIsolationMode=Merged
в папке, в которой нужно что-либо реально изменять (вместо Merged можно Full, открывает полный доступ для программы). Например, для сохранения файлов на рабочем столе в компилируемых папках папка %Desktop% должна содержать этот файл.
2. Перед компиляцией в Package.ini пометить как комментарий старый параметр и убрать пометку с нужного нам. Чтоб было понятнее, было так:
[Isolation]
DirectoryIsolationMode=WriteCopy
;DirectoryIsolationMode=Merged
стало так:
[Isolation]
;DirectoryIsolationMode=WriteCopy
DirectoryIsolationMode=Merged

Извиняюсь, дал неверную информацию. Full режим в Isolation самый жесткий:
- System elements at this location will not be visible to application
- modifications to virtual elements go to sand box
- system elements cannot be read or modified
- New elements will be created in the sandbox
по-русски:
- Системные элементы в этом местоположении не будут видны приложению
- модификации к виртуальным элементам идут в песочницу (прим. sandbox - песочница с англ.)
- системные элементы не могут быть прочитаны или изменены
- Новые элементы будут созданы в песочнице
