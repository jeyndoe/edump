защита открывает исполняемый файл на диске, читает его кусками по 2 кила и вычисляет проверочную комбинацию. есть относительно простой способ обойти такую проверку целостности. т.к. защита открывает файл с диска, то имя этого файла должно либо храниться в проге, либо получаться в процессе выполнения. в тотале оно получается с помощью Win32 API функции GetModuleFileNameA. Поэтому после вызова данной функции перед вызовом CreateFileA можно сделать инлайн и изменить имя файла. Например изменив одну букву в имени и скопировав оригинальный тотал в файл с таким именем. Таким образом исполняться будет один файл, а защита будет проверять целостность другого файла.
