hi,

Your vendor name is daemon name:

INCREMENT test_package test_vendor 1.0 permanent uncounted

If you make your lmcrypt with flexnet 11.4 with fished vendor keys and encryption seeds (maybe cro keys also) is posible to start with lmgrd.exe your test_vendor.exe generated daemon file and with license file generated with
Builded lmcrypt.
Most important is to add 3 strings:

SERVER your_comp_name mac_address 756@your_comp_name
VENDOR test_vendor
USE_SERVER
#....
INCREMENT test_package test_vendor 1.0 permanent uncounted




With lmcryptgu.exe you generate only lmcrypt.exe and you need for serv lic daemon.exe
But if you use this method its is posible very simply to find hidden features from debug.log file
