Хотелось бы подвести небольшой итог исследования работы Page Promoter начиная с версии 7. В ходе нескольких экспериментов выяснилось примерно следующее:

Ресурсы программы шифруются. Причем для каждого легального пользователя каждой версии и Demo версий ключи разные. Поэтому полная база newengine.pdb легального пользователя не может быть расшифрована ключем Demo версии и не работает. Шифрование происходит предположительно по алгоритму blowfish.

На сервере для каждого пользователя создается свой каталог с обновлениями, зашифрованными соответствующим ключем пользователя и каталог для Demo версии. Зашифрованный путь к обновлениям выглядит примерно так:
825043C1C4BE5685BFDA79F48FD640B7FD74773FB0724F8C05982B738F4CD24C

Например для Demo версии 7.2 запрос на обновление баз к серверу выглядит следующим образом:
www.net-promoter.com/reg/825043C1C4BE5685BFDA79F48FD640B7FD74773FB0724F8C05982B738F4CD24C/database/engines.zip и скачивается файлик register/database/PagePromoter/7.2/Demo/engines.zip

Т.е. в 825043C1C4BE5685BFDA79F48FD640B7FD74773FB0724F8C05982B738F4CD24C зашифрован путь register/database/PagePromoter/7.2/Demo

Для каждого легального пользователя соответственно разный зашифрованный путь.

В файле engines.zip содержатся и обновляются файлы:
db\addfunc.pdb
db\newengine.pdb
db\engrank.pdb
db\linkage2.pdb
db\search2.pdb
db\class.pdb
db\allimg.pdb
db\Endef_cat.pdb
db\def_cat.pdb
db\Rudef_cat.pdb
db\cat_wordlist.pdb
rudeslingua.ini
RULingua.ini
flank.pdb
db\gdance.pdb
db\gddata.pdb
db\kb.pdb
db\kbgroups.pdb
db\popularity.pdb
db\dcmes.pdb

Можно, конечно, попробовать достать файлик engines.zip легального пользователя, но думаю, что ключ Demo версии все равно не сможет дешифровать ресурсы из него. А при копировании и использовании вместе с engines.zip ключа легального пользователя, он скорее всего сразу забанится.
