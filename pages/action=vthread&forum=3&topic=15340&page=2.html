What's new in v.2.0.1.48:
~~~~~~~~~~~~~~~~~~
- Completely re wrote multi assembler. aka ROTE aka Rainbow Ollydbg Text Assembler
- New Goto dialogbox with code completion
- A lot of new shortcuts

[+]: Added mouse scrolling ability to Goto listbox.
[+]: Added a workaround StrongOD problem hooking NtCreateProcess: You have to
go to settings>rote tab, then set the number of milliseconds to sleep
after compilation (500 ms works fine form me on a core 2 duo 1.60GHz).
Thus, I noticed that it succeed only if THERE ARE NO ERRORS in the code.
[+]: Ability to supersede StrongOD and AdvancedOlly Goto dialogbox even if
not disabled ('Choosable').
[+]: Ability to supersede StrongOD status bar info (idaficator must be loaded
last, and to do so, just execute ollydbg without idaficator, then add it
and re-run (after checking 'Supremacy mode' from settings)).

XRipper v0.8 Stable, Build 726

Here is a space for a stable professional release
after a decade of no activity , have fun!

XRipper v0.8 Stable, Build 726 {26-07-2011}:

What is it?!

XRipper is a tool that allows you to scan and extract hidden files
(xm, v2m, png, bmp, raw, ogg, swf, ttf, mo3, it, wav, spc, mid, jpg, rmi,
iff, aif, flc, mod, s3m, mtm, umx, bnk, 669, voc, emf, sid, fc, zip and nsf)
Stored inside Games, Keygens, Patches, Intro's, Archives and more..

And what's new?

[\o/] XRipper Core Engine have been Enhanced (Should be more faster).
[+] Added Turkish Language (Thanks to C0dr4cK ^ TccT).
[+] Added Support For : MID, JPG, RMI, MOD, S3M, MTM, UMX (Play/Preview & Ripping),
IFF, AIF, FLC, BNK, 669, VOC, EMF, SID, FC, ZIP And NSF (Only Ripping).
[+] Added Detection For:
SQLite, XML, HTML, DOC, PPT, 7z, RMI, FLAC, XI, DB,
3GP, AVI, PF, RMVB, FLV, GZ, TAR, TTA, DCU, XRS, DAT,
DPR, OBJ, DBB, VDI, KEY, TGA, ICO, APE, SPX, MPC, WV,
RM, IFF, AIF, FLC, UMX, BNK, 669, VOC, UPX, FSG ..
[+] Added FileDetector.dll v0.8 Build 726 (SDK\File Detector).
[+] Added Internal Plugin Loader For:
PEiD Plugins Loader.
QUnpack Plugins Loader.
PETools Plugins Loader.
[+] Added Built-In Hex Editor.
[+] Added ZipView (Written from scratch).
[+] Added Macros for file saving: %offset%, %extension%.
[+] Added Special Sound Effects on Application Events.
[+] Added Hide to Tray (TrayIcon).
[+] Added Always on Top Option.

[+] Added Option To Scan from Specific Offset.
[+] Added Option To Dump Process Memory To File.
[+] Added More Options to User Custom Formats.
[+] Added Extra Options to Image Preview (Brightness, Contrast).
[+] Added Option To Save Found Files As: Delphi Unit & Include (.pas,.inc), Visual C Header (.h), Masm Include (.inc).
[+] Added Option To Scan a Specific Process Memory Range (16 Bit, 32 Bit or All Memory).
[+] Added Full Clipboard Formats Scan.

[+] Added Playing OGG Before Ripping Option.
[+] Added Enumerate False Detections (Weak file signatures are excluded).
[+] Handling Multiple List Item at once.

[>] Discover the rest..

[-] Playing SWF's Enhanced (No more Extract Play&Delete).

[-] New Look, Options rearranged, easier and more user friendly.

Fixed Bugs:
[x] Fixed MO3 False Detection.
[x] Fixed Multi-Parts Ripping for same OGG file.
[x] Fixed bug with some English phrases was not included in translation.

- Size of ripped wav files was missing 16 Bytes at the end of file (EOF).
- Ripping two waves with same Offset cause creating 1st file and erasing it with 2nd.
[x] Fixed bug with Flash Player: After closing Flash Form, XR continious playing it in the background.

[x] Fixed some false files detection.
[x] Fixed Memory suffer when finding alot of PNG's.
[x] Freeing some components wasnt included in freeing list (SplachScreen..).

Exclamations:
[!] - Using INI files to hold settings is not safe since the user can to modify the file contenents,
So i moved to use a binary file to save settings in it,
the file have extension XRS (XRipper Settings File),
have fun and dont modify or delete it or all your settings will be lost!!.
- Format INI Changed To XRipper Format File (XRF) same issue as above.
[!] From now XRipper.exe is unpacked due to a png compression issue.
[!] Working on new XRipper Engine, Extreemly faster!! (10x), but probably it will
be fully implemented and released in version (9.. Who Knows!) as Generic Search.
[!] Many Options are hidden because unfinished and i didnt want to take longer time to release this version.

Unded Developement:[*] User Detection Signatures.[*] Search for Files Using Format DLL Plugins (BMP.dll ..).[*] New Type of XRipper Plugins called Extended Plugins.[*] XRipper Lite Universal Plugin.

In Progress:

[\/] Search & Extract For ICO and ANI Files.

Release Hints & Tips:
[?] Using "Scan Inside Detection" Option will Increase search average time by few seconds
depends on the file size and the number of extensions you choosed to search for.
[?] - Using "Override Length Overflow" Option may not always should give the good results,
Uncheck this option if you think that the found file has unapropriate size and try to search again.
- Also this otpion wont work if there is only one found file, why? because SHR in Math need more
Than one integer!, so we need file after the first file so XR will count the size of first file
by counting: File1Size = Start offset of File2 - Start offset of File1.
