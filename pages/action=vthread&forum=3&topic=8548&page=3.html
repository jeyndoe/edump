Code:
  1. - Fixed a number of cases when a Syntax Highlighter was not properly applied when a file was opened.
  2. - Fixed an error running a Syntax Highlighter when the first line of a file was blank.
  3. - Fixed using 'return' inside a struct which was declared in a function would cause an error in the function.
  4. - Fixed bookmarks created while the Bookmarks panel was hidden were not properly shown when the Boomkarks panel was displayed.
  5. - Fixed on Linux when using a dark theme, the category headings in the Repository dialog could not be read.
  6. - Fixed the 'Hide Characters over 127' toggle should always be turned off when a custom character set is created.
  7. - Fixed passing a very long line to a Syntax Highlighter could cause the software to slow down. Syntax Highlighters now only color up to the Maximum Line Length as set in the Text Editor Options.
  8. - Fixed a problem running a template on a template, or a script on a script using the File Bar (right-click on the File Bar to enable this).
  9. - Fixed the Sleep function on Linux/macOS was using microseconds instead of milliseconds.
  10. - Fixed the Variables tab was not always showing the results after running a script.
  11. - Added a warning when a very long line was encountered (over 100,000 bytes) and word wrap was turned on. These lines cannot currently be fully wrapped and this will be improved in the future.

SweetScape 010 Editor v9.0.2 (26.04.2019)
Code:
  1. Fixed icon transparency for the Windows Explorer shell extension.
  2. Fixed syntax highlighting for keywords (HighlightMatchKeyword) now finds the longest matching keyword instead of the shortest (e.g. 'background-repeat' vs 'background').
  3. Fixed when 010 Editor is run as administrator on Windows, files could no longer be opened from the Windows Explorer shell extension. Now a new non-admin copy of 010 Editor is opened before the files are opened.
  4. Fixed on Linux the IBus IME can now be used.
  5. Fixed possible crash in Memcpy, StrCat and SubStr when using very large arrays.
  6. Fixed possible memory overrun when downloading news.
  7. Fixed when opening a text file from the command line containing a BOM, sometimes bytes were improperly selected at the beginning of the file
