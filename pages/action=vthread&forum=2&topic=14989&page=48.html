Для п.5 (см. ниже) необходимо указать компилятор(протектор) Вашей программы, который можно определить с помощью анализатора PEiD. БЕЗ ЭТОГО ПУНКТА ЗАПРОС БУДЕТ УДАЛЕН.
Причем для PEiD надо указывать не файл установщика (setup.exe, и т.д.), а сам исполняемый файл, который вы запускаете для работы с программой! Исключение - программа не может быть установлена без валидного серийника, т.е. взламывать необходимо сам инсталлятор.

Требуется разобраться с одним алгоритмом в программе, т.е. это не совсем взлом.

Есть программа, написанная на Delphi 7, эта программа является десктопным клиентом к Веб-Сервису.
Сервис защищен, у меня есть логин и пароль для входа. Т.е. я легально могу работать через этого клиента.

Но у меня стоит задача написать программу, которая бы заменила этого декстопного клиента.

В сервис для авторизации надо отдать token вида:
<wsse:Security>
<wsse:UsernameToken>
<wsse:Username>name</wsse:Username>
<wsse:Password>BNNNbSN1qAcbDPMh4ubCM3+r8GI=</wsse:Password>
<wsse:Nonce>M09TfCteeExKKndecWg3VyRWbC1dWC9mcVRmcShoS1c=</wsse:Nonce>
<wsu:Created>2010-06-18T12:37:05Z</wsu:Created>
</wsse:UsernameToken>
</wsse:Security>

Генерирую образ пароля по спецификации: pass_dig = base64(sha1(nonce + created + password)) - не совпадает с тем, что посылает программа в сервис.

У меня два предположения:

1. Используются алгоритмы формирования пароля отличные от спецификации WS-Security.
2. Перед отправкой в сервис пароль, который я ввожу в систему, изменяется (дополняется).

Помогите разобраться с алгоритмом формирования образа пароля.
