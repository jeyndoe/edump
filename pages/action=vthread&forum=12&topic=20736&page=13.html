Предлагаю дополнить сигнатуру определения библиотеки Qt условием поиска признака статической линковки. Многие разработчики помимо динамической линковки Qt, когда а каталоге установке софта находятся файлы QtCore.dll, QtGui.dll и т.д., используют в своей практике статическую линковку Qt, когда необходимые приложению библиотеки Qt зашиты в исполняемом файле. Незнание этого факта обламывает любые попытки локализации подобного софта на русский язык.

В файле сигнатуры QT.3.sg нужно дописать следующее условие (перед командой return result):
Code:
  1.     if(!aQt&&PE.section[".rdata"])
  2.     {
  3.         var nVersionOffset=PE.findString(PE.section[".rdata"].FileOffset,PE.section[".rdata"].FileSize,"QWidget");
  4.         if(nVersionOffset!=-1)
  5.         {
  6.             sVersion="x.xx";
  7.             sOptions="static linked";
  8.             bDetected=1;
  9.         }
  10.     }


Пример, программа Batch Imager (V-Methods Software).
