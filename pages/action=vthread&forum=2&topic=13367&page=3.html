Code:
  1. A. The quadratic sieve really is not appropriate for factoring numbers
  2.    over ~110 digits in size. On a fast modern CPU, a 110-digit QS factor-
  3.    ization takes nearly 120 hours for Msieve, and the time increases
  4.    steeply beyond that. If you have really big factorization needs, there
  5.    are several packages that you can use to make up for the low-performance
  6.    parts of Msieve's NFS code:
  7.  
  8.          - CADO-NFS (cado-nfs.gforge.inria.fr) is a complete NFS suite
  9.            that includes state-of-the-art (or nearly so) implementations 
  10.            of all phases of the Number Field Sieve. The sieving tools in 
  11.            CADO-NFS produce output compatible with GGNFS and Msieve, so it's
  12.            up to you which package you want to use for a given phase of NFS.
  13.            In addition, CADO-NFS is under constant very active development,
  14.            and in my opinion represents the future of the high-performance
  15.            NFS community.
  16.  
  17.          - GGNFS (www.sf.net/projects/ggnfs) has state-of-the-art tools
  18.           for NFS sieving and NFS polynomial selection. Using the latter 
  19.            is optional, since Msieve actually has very good NFS polynomial
  20.            selection code, but the former is an absolute must if you 
  21.            are factoring numbers larger than about 95 digits.
