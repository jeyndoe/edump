мало того, что TrashReg меньше весит (т.к. VB), так ещё и удаляет всю дрянь, в т.ч. и последних версий аспра и армы, в отличие от упомянутой mysterio проги

по поводу неудаления некоторых ключей реестра, то скорее всего, эти ключи/разделы содержат символ конца строки (0) в имени, такие ключи/разделы стандартные API не удаляют (т.к. для них 0 - конец строки)
нужно, соответсвенно, использовать API, не обращающие внимание на наличие нуля в имени (длина имени передаётся в кач-ве параметра и не вычисляется по наличию символа конца строки)

Registry Trash Keys Finder, версия 3.7.0 SR1

v.3.7.0 (24.11.2005)

Поисковое ядро:
- добавлено: поиск ключей защиты ExeCryptor;
- добавлено: поиск ключей защиты ZipWorx;
- добавлено: поиск ключей защиты SDProtector;
- добавлено: поиск ключей защиты Trial Creator;
- добавлено: поиск пуcтых ключей TypeLib;
- небольшие корректировки в алгоритме поиска ключей,
созданных последними версиями ASProtect;
- теперь возможно удаление "Скрытых ключей", содержащих
любое количество вложенных ключей, причем последние
в свою очередь тоже могут быть "скрытыми";
- теперь возможно извлечение полной информации о
содержимом найденного "скрытого" ключа.

Интерфейс:
- добавлен Немецкий интерфейс (перевод - Dirky);
- добавлено "классическое" меню с несколько
расширенным набором команд;
- добавлено меню окна просмотра найденного ключа;
- добавлена справка в формате HTML (chm)
(русская и английская версии);
- теперь всплывающая подсказка с информацией
о ключе несколько более подробна и отключаема;
- теперь функция "Открыть в Regedit" работает очень
быстро и без мельканий на экране монитора;
- некоторые изменения в окне настроек программы.

Командная строка:
- /j=путь_к_ключу_реестра для быстрого открытия
указанного пути в Реестре

Багфиксы:
- множество.
