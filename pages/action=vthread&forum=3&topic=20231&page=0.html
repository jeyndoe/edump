GSS Visual Assembler (последняя на данный момент версия 3.9.8)

GSS Visual Assembler — среда разработки программного обеспечения для ОС Windows, предназначенная для разработки программ на языке ассемблера.

Удобная визуальная оболочка с легко настраиваемым интерфейсом напоминающем интерфейс Borland Delphi или Microsoft Visual Studio.
Изначально создавалась под такие ассемблерные пакеты как TASM и MASM32 но есть возможность адаптировать IDE и под другие пакеты.
Основные характеристики:
- Возможность настроить IDE практически для любого Ассемблерного пакета (MASM32, TASM и прочих).
- Поддерживается интегрированный отладчик под MASM32, позволяющий отлаживать
программы прямо по исходному коду (без дизассемблирования).
- Настройка любой схемы подсветки текста.
- Отображение полной структуры кода (процедуры, макросы, структуры, константы и т.д.)
- Автоматический поиск декларации переменных.
- Добавление ресурсов к проекту.
- Максимальная простота управления проектом.
- Пользовательский интерфейс максимально копирует привычный интерфейс Microsoft Visual Studio или Borland Delphi.
- Поддержка всех версий платформы Windows.

GSS Visual Assembler разработан с учетом потребностей, как профессиональных разработчиков, так и начинающих.

Чем отличается GSS Visual Assembler 3.9.8 от 2.10.9?
- Полностью переработанна система распознавания структуры кода, благодаря чему удалось существенно дополнить функционал программы.
- Практически полностью переработанный редактор.
- Показ параметров WinAPI процедур при вводе директивы "invoke".
- Показ списка используемых прототипов при вводе директивы "invoke".
- Поле "Format" главного меню.
- Переработаны все окна программы.
- Более надёжная работа.
- Поддержка интегрированного отладчика под MASM32.
Отладчик позволяет отлаживать 32-bit многонитевые приложения прямо по исходному коду.
Возможности отладчика на данный момент: быстрая расстановка брэйкпоинтов по исходному коду,
просмотр/модификация контекста текущей нити, просмотр/модификация стэка текущей нити, просмотр/управление нитями.
Отладка DLL модулей.

Что нового в GSS Visual Assembler 3.9.8?
- Исправлена серьёзная ошибка возникавшая при перерисовке редактора.
- Исправлены ошибки открытия и закрытия файлов.
- Исправлена ошибка иногда возникавшая при первом открытии окна "CodeComplation".
- Исправлены серьезные ошибки при синхронизации потоков.
- Исправлены множество мелких (и не очень) недоработок что сделало 3.9.5 гораздо
более стабильней и удобнее в работе, чем 3.9.
- Обновлен интерфейс

- Улучшен "интеллект" программы.
- Появилась возможность показа параметров структуры которая ассоциирована с каким либо процессорным регистром.
К примеру если обращение к какой-либо структуре выражено как "[eax]." или "(STARTUPINFO ptr [eax])." то
будет автоматически определенно с какой именно структурой ассоциирован данный регистр и соответственно
её параметры будут показаны в окне "CodeComplation".
- Появилась возможность показа параметров прототипов функций не только WinAPI но и объявленных пользователем.
- Показ параметров прототипов функций возможен также если пользователь набирает их в многострочном варианте.
- Больше нет ограничений на длину названий процедур, макросов, прототипов и т.д. (которая раньше составляла 256 символов).
- Добавлено несколько полезных приёмов которые упрощают редактирование кода.
- Упрощенно составление командных строк для ассемблерных трансляторов. Теперь её можно составлять в многострочном поле.
